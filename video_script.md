# EventSphere - College Event Management System Demo Script

## Introduction (30 seconds)
"Hello everyone! Welcome to EventSphere, a comprehensive College Event Management System. Today I'll be demonstrating the key features of this platform that helps students discover, register for, and manage college events seamlessly. Let's dive in!"

## Homepage Tour (45 seconds)
"First, let's look at our homepage. As you can see, we have a clean, modern interface with the EventSphere branding. The navigation bar includes Home, Events, Gallery, About, Contact, and Dashboard sections. The homepage features upcoming events, recent activities, and quick access to popular events."

*Navigate through the homepage, showing the layout and design*

## User Registration/Login (1 minute)
"Let's start by creating a new account. I'll click on the Login/Register button."

*Click on login/register*

"Here we have our registration form where users can sign up with their email, password, and basic information. The system supports role-based access - students can register as participants, while staff can have organizer or admin roles."

*Show registration form, fill it out*

"For existing users, the login process is straightforward - just enter your email and password."

*Demonstrate login process*

## Events Browsing (1.5 minutes)
"Now let's explore the Events section. Here you can see all available events with detailed information."

*Navigate to Events page*

"Each event displays:
- Event title and description
- Date and time
- Venue location
- Category (Technical, Cultural, Sports, etc.)
- Registration status and availability
- Event organizer information"

*Scroll through events, point out different categories*

"Users can filter events by category, date, or search for specific events. Let's look at a technical event - 'Techwaiz'."

*Click on an event to show details*

## Event Registration (1 minute)
"When you find an interesting event, registration is simple. Click the 'Register Now' button."

*Click register button*

"The system checks availability and confirms your registration. You'll receive a confirmation with your unique registration ID. The system also tracks registration status - confirmed, waitlist, or cancelled."

*Show registration confirmation*

## Dashboard Overview (2 minutes)
"Let's check out the user dashboard - this is the heart of the user experience."

*Navigate to dashboard*

"The dashboard provides a comprehensive overview:
- Welcome message with personalized greeting
- Statistics showing registered events, attended events, certificates earned, and upcoming events
- My Registered Events section with all your current registrations
- Recent Activity feed showing your latest actions
- My Certificates section for earned certificates"

*Point to each section while explaining*

"In the 'My Registered Events' section, you can see all events you've registered for, including:
- Event details like date, time, and venue
- Registration status (Approved, Pending, etc.)
- Action buttons to view details or download QR codes"

## QR Code Generation (1 minute)
"One of the standout features is the QR code generation for registered events."

*Click on a registered event*

"Each confirmed registration generates a unique QR code that serves as your digital ticket. You can download this QR code for easy check-in at events."

*Show QR code download process*

"The QR code contains all necessary information - your registration details, event information, and verification data. This makes event check-in fast and efficient."

## Additional Tips for Recording:

### Before Recording:
1. **Prepare test data**: Ensure you have sample events, users, and registrations
2. **Clear browser cache**: Start with a clean session
3. **Check all features**: Test QR code generation, downloads, etc.
4. **Prepare multiple user accounts**: Show different user roles if possible

### During Recording:
1. **Speak clearly and at moderate pace**
2. **Use cursor highlighting** to point to specific elements
3. **Pause briefly** between sections
4. **Show actual functionality** - don't just describe it
5. **Handle errors gracefully** if they occur

### Recording Settings:
- **Resolution**: 1920x1080 (Full HD)
- **Frame rate**: 30 FPS
- **Audio quality**: Clear, no background noise
- **Screen area**: Focus on browser window
- **Duration**: Aim for 8-10 minutes total

### Post-Recording:
1. **Add captions** for accessibility
2. **Include timestamps** in description
3. **Add intro/outro graphics** if desired
4. **Export in high quality** for professional presentation
